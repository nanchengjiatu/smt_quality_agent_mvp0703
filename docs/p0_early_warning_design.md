# P0 事前预警设计:EWMA 漂移监控 + 过程能力

状态:2026-07-06 草案,待评审。评审通过前不动代码。

## 1. 目标与定位

把产品从"NG 之后分析根因"翻转为"NG 之前拦截漂移"。SPI 判 NG 是结果,
锡膏印刷的多数渐变型失效(锡膏流变性漂移、网底残留累积、擦网周期不匹配)
在判 NG 前会先表现为连续测量偏差的爬升——这部分信号目前完全未被利用:
当前实时表 46,587 行里 46,575 行是 PASS(99.97%),每行都带 Pad 级
体积/面积/高度偏差幅度(Comp_avdp/aadp/ahdp)。

预警**不是判定**:它不进 NG 口径、不进三板连发触发、不改变现有决策梯,
只作为独立的前瞻信号层,提示"这个 Pad/这块钢网正在变坏,越限之前去看一眼"。

### 1.1 价值主张两条腿(应对 §2.7 的探针结果)

1. **漂移预警**(渐变型 NG 的提前拦截):这是长期主张,但当前数据里的
   NG 全呈突发型,这条腿的验收要等回测/新数据说话;
2. **Pad 健康度监控**(不依赖"NG 提前量"就成立):92 Pad 的基线水位、
   波动、漂移裕度一屏可见——探针顺带发现触发 Pad C1754_2 的基线
   μ=35.8/σ=10.5,长期显著高于全体均值 18.9,这种"慢性高偏差 Pad"
   正是 margin 视图(§3.4)要暴露的对象,今天的数据就能交付价值。

## 2. 关键数据事实(决定方案走向)

以当前实时表 full_excel0625 为准(单机种 C1027835AB13,1659 块板,92 个
Pad,每板每 Pad 一行;36 块板有复测):

1. **Comp_\*dp 是无符号偏差幅度**(全表最小 0,方向只在 errName 里)。
   → 监控只能做**单边上限**:幅度爬升 = 变坏,不区分多锡/少锡方向;
   方向在预警阶段未知,机理提名按"双向"投影,等出现 NG 再收敛方向。
2. **PASS 与 NG 的偏差分布重叠,表里没有规格限列**:PASS 行 avdp
   均值 18.9 / σ 11.1 / 最大 100;而 Under Volume 行 avdp 最低只有 27.9。
   SPI 的判定阈值不在导出数据里,也不是单纯按偏差幅度判的。
   → **真 Cpk 缺 USL,第一期不硬造**。过程能力先用"漂移裕度"表达
   (见 §3.4),真 Cpk 等 SPI 判定阈值确认后启用(数据侧需求,§7)。
3. **每 Pad 序列足够长**:92 个 Pad 各约 1600 点,EWMA 收敛(几十板)
   和滚动基线(200 板)都绰绰有余;滑窗 3000 板远大于所需回溯。
4. **回测锚点现成**:两个真实三板连发触发(TRG-d1489186 多锡 /
   TRG-e1f5edac 少锡)+ 12 条 NG,可以量化"提前几块板报警"。
5. **现有 precursor 判据可直接翻转**:param_correlation 的事件回溯
   (回看 20 块正常板,斜率 ≥0.5%/板 且末段连升 ≥3 板判"有前兆")
   已在真实事件上验证过"渐变型存在可预警窗口",P0 把同一判据从
   "事后回看"改为"逐板前瞻"。
6. 机理层已预留挂点:13 个 FailureMechanism 中 4 个标了 early_warning
   (锡膏流变性漂移=整板均值 EWMA、网底残留=擦网间隔内偏差递增、
   密合不良=高度/面积反向漂移、擦网周期不匹配=周期检测+频率参照),
   预警的解释层不需要新知识,复用 v4/v5 决策管道。
7. **⚠ 微型探针结果(2026-07-06,写设计前实测)**:当前表 6 个事件在
   整板均值口径下全部判"突发型";对两个真实触发 Pad 做 Pad 级 EWMA
   粗探(λ=0.2/L=3),越限点在首块 NG 前 150 / 641 块板——不是临近
   前兆,而是这两个 Pad 基线波动本身偏大;NG 前末 10 板均无单调爬升。
   即**现有数据里的 NG 在 Pad 级同样呈突发型**。设备参数列全程贴
   计划值无漂移,与突发结论互洽(更像异物/气泡类瞬时因素)。
   → 结论:在这份数据上,"提前 N 板拦截 NG"的验收指标可能无法达成。
   这不推翻 P0(渐变机理真实存在,只是这批数据没赶上),但要求:
   ① 回测先行的阶段划分不可省略;② 价值主张两条腿走(§1.1)。

## 3. 方案:三层结构(监控 → 分级 → 解释)

### 3.1 监控层:三条相互独立的控制线

全部纯 stdlib、幂等重算(与滑窗 pipeline 同一模式:每轮全窗重算,
不维护增量状态机)。输入一律为**首检行**(复测剔除,与现有口径一致)。

1. **Pad 级 EWMA**(主力):对每个 Pad × 每个指标(avdp/aadp/ahdp),
   - 基线:滚动前 200 块板中该 Pad 的 PASS 行 μ/σ(**报警期间冻结**,
     防止漂移污染基线,漂平后解冻);
   - EWMA:z_i = λ·x_i + (1−λ)·z_{i−1},λ 初值 0.2;
   - 越限:z_i > μ + L·σ·√(λ/(2−λ)),L 初值 3,只看上限。
   λ 和 L 不拍脑袋定死,由回测标定(§5)。
2. **整板级 EWMA**:板内 92 Pad 偏差均值的 EWMA,同样参数体系。
   捕捉锡膏整体流变性漂移(单 Pad 线看不出的全局缓漂)。
3. **擦网周期参照线**:距上次清洗的板数 vs 板均偏差的锯齿形态
   (复用 v3 第二阶段的擦网频率参照逻辑):锯齿振幅逐周期增大、
   或周期尾部偏差逼近越限,是网底残留/擦网周期不匹配的专属前兆。

### 3.2 分级层:预警阶梯(与决策梯风格一致,但互不混入)

| 级别 | 判据 | 语义 |
|---|---|---|
| L1 观察 | 单指标 EWMA 越限 1 板 | 记录,不打扰 |
| L2 预警 | 连续 ≥3 板越限,或 ≥2 个指标同板共振越限 | 页面提示,建议关注 |
| L3 高危 | L2 且末段连升 ≥3 板(等价既有 precursor 判据),或实测值进入历史 NG 观测区间(avdp ≥ 27.9) | 显著告警,建议干预 |

阈值取 3 板对齐三板连发的严格连续口径,语义好解释:
"再这样漂 3 块板,可能就是一个真触发"。

### 3.3 解释层:机理提名(复用,不新造)

预警产生时,用已越限指标构成三指标签名(哪些指标在漂、哪些稳定),
经 project_mechanisms 投影——但**只投影 early_warning 非空的 4 个机理**,
方向按"双向"(§2.1)。输出与下钻一致的候选结构(机理/置信/建议动作),
走同一套置信度阶梯,decision_trace 照发。预警卡片因此天然带
"为什么报、建议做什么",而不是裸曲线。

warning_id 与 trigger_id 同方案:`WRN-<sha1(model|pad|首次越限板barcode|时间)[:8]>`,
跨重算稳定,支持已读/确认延续;整板级预警 pad 位取 `__board__`。

### 3.4 过程能力:第一期用"漂移裕度",真 Cpk 挂起

每 Pad 输出 margin = (历史 NG 观测下界 − 当前 EWMA) / (3·σ_基线)。
margin 是"按当前波动水平,距离出过 NG 的偏差水位还有几个 3σ",
语义接近 Cpk 但诚实标注**参照的是观测边界,不是规格限**。
SPI 判定阈值到位后,同一公式换成真 USL 即为 Cpk,前端字段不变。

## 4. 输出与 UI

- pipeline 新增第 4 个 stage `early_warning` → `output/early_warning.json`:
  `{ generated_at, params: {lambda, L, baseline_boards}, warnings: [ {warning_id,
  level, pad_name|__board__, metrics, ewma_series(截近 N 板), threshold,
  first_exceed_time, mechanism_candidates, margin, status(active/recovered)} ],
  pad_health: [ {pad_name, ewma, margin, level} ×92 ] }`。
  聚合失败与其他 stage 一样单独报错,不拖垮别的视图。
- 前端新页签**「事前预警」**:上半区 92 Pad 健康矩阵(按 margin 着色,
  一眼看哪些 Pad 在变坏),下半区活动预警卡片(EWMA 曲线+越限带+机理
  候选+建议动作)。诚实空态:"当前无漂移预警" 而不是空白。
- dashboard 指标区加「活动预警」tile;自动更新出现新 warning_id 时
  toast 提示(优先级低于新触发,高于普通数据更新)。
- 预警后来真的升级成三板连发触发时,触发卡片回链 warning_id
  ("此触发曾于 N 板前被预警")——这是产品定位翻转的直接证据,
  也是 P3 知识闭环的天然素材。

## 5. 回测:阈值用数据定,验收有硬指标

实施第一步不是接 pipeline,而是先写回放 harness:按 fdate 序逐板重放
1659 块板,输出每次预警的(pad, 级别, 时刻)。用它:

1. **标定 λ/L**:扫参数网格(λ ∈ {0.1,0.2,0.3} × L ∈ {2.5,3,3.5}),
   在"对两个真实触发的提前量"与"误报率"之间取折中;
2. **验收指标**(写进测试,不达标不上线):
   - 渐变型事件提前量:对确认有前兆的事件,L2 预警须早于首块 NG ≥3 块板;
   - 误报预算:L2 及以上 ≤ 1 次 / 100 板(全 92 Pad 合计,待拍板,§7);
   - 突发型事件如实不报(现有 param_analysis 已判定哪些事件是突发型,
     控制图本来就拦不住,预警系统不为此背误报)。
3. 回测报告(各参数组合的提前量/误报表)先出给用户评审,再定参数。

§2.7 的微型探针就是这个 harness 的最小雏形;阶段 1 把它正式化
(全 92 Pad × 全参数网格 × 报警持续性/恢复判定),并回答探针遗留的
问题:远早于 NG 的越限,应算"慢性高波动 Pad 的健康度问题"(L1 记录、
计入 margin)还是误报(计入误报预算)——这决定 L2 的"连续 ≥3 板"
门槛是否要再加"相对基线抬升幅度"条件。

## 6. 不做(本期非目标)

- CUSUM / 多变量 T² / 机器学习:EWMA 不够用再说,单机种 12 NG 的数据量
  养不起学习型模型;CUSUM 留作 λ 调不平衡时的备选。
- 逐板流式推送:预警随每轮滑窗重算产出(数据速率 ~45 板/天,轮询足够)。
- 短信/邮件/外部通知:页面内提示;通知渠道等真实用户提需求。
- 有符号方向监控:数据不含方向(§2.1),不造假信号。

## 7. 决策点(评审时请拍板)

1. **误报预算**:L2 及以上多少算可接受?建议先按 ≤1 次/100 板标定,
   回测报告出来后按真实曲线再调。
2. **L3 的"NG 观测区间"判据**:用历史 NG 最低 avdp(27.9)做参照线
   是否合适?它会随新 NG 数据自动收紧/放宽(取滑窗内 NG 最小值)。
3. **页签命名与位置**:「事前预警」放实时异常左侧(第一位)还是右侧?
   放第一位等于宣告产品定位翻转。
4. **数据侧新需求(加入既有清单)**:SPI 判定阈值(每 Pad 或每机种的
   体积/面积/高度上下限)——这是真 Cpk 的前提,建议向 SPI 机台/产线
   索取判定配置导出。

## 8. 阶段划分

| 阶段 | 内容 | 出口 |
|---|---|---|
| 1 | EWMA/基线/分级纯函数 + 回放 harness + 参数网格回测 | 回测报告供评审,定 λ/L |
| 2 | early_warning stage + JSON 契约 + 机理提名接线 + 测试 | output/early_warning.json 落盘 |
| 3 | 前端页签(健康矩阵+预警卡)+ dashboard tile + toast + 触发回链 | 页面可验收 |
| 数据侧 | SPI 判定阈值导出(真 Cpk 前提) | 到位后 margin 换真 USL |

阶段 1 先行且独立——回测结果不理想(比如两个真实触发都是突发型、
提前量为 0)时,方案在动 pipeline 之前就能低成本转向。
