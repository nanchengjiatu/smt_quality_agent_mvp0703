# P1 根因维度补齐:工艺维度体检

状态:2026-07-06 设计并实施。前置探针结论直接塑造了产品形态(见 §2)。

## 1. 目标

路线图 P1 的三个根因维度——钢网清洗周期效应、停机后首板效应、印刷方向
奇偶差异——补齐为**持续计算的"工艺维度体检"**:每轮 pipeline 随
param_analysis 重算,在"事件与根因分析"页以三张体检卡呈现,每张卡如实
报告四种状态之一:存在效应 / 无明显效应 / 样本不足 / 数据未采集。

## 2. 探针结论(2026-07-06,决定形态)

对 full_excel0625(1659 板/37 天)实测:

1. **清洗周期**:cleaningfrequency 恒为 3(每 3 板擦一次网,周期极短),
   按周期折叠的板均偏差曲线振幅 0.47pp,对比板均噪声 σ=5.77pp——
   **无周期效应**。
2. **停机首板**:≥30 分钟停机 37 天仅 5 次(产线近连续运行),首板偏差
   均值反而更低(15.3 vs 18.8)、零 NG——**无不利效应且样本不足**。
3. **方向奇偶**:Front/RearSQGPress 恒为 4.0、printmode 恒为 Once,
   PrintDirection 列不存在——**数据不含方向信号**,奇偶差 0.34pp 无意义。

→ 三个维度在这份数据上都"没有故事"。因此 P1 的价值不是今天的结论,
而是**体检机制本身**:换机种/换产线/数据列到位后,效应自动点亮;
在此之前页面如实显示灰色状态,不编造根因维度。这与知识模型 v3 的
"auto 证据带 availability、未采集如实说"是同一原则。

## 3. 实现

- 新模块 `smt_quality_agent/process_dimensions.py`(纯函数,零依赖):
  - `analyze_cleaning_cycle`:按 cleaningfrequency 折叠板均 avdp,
    输出逐位置 profile、振幅、噪声参照;判效应门槛 =
    max(1pp, 0.5×板均σ)。诚实前提:清洗时刻未记录
    (CleaningAfterLastBoard 未采集),折叠假设节拍从序列起点稳定,
    相位未知——卡上如实标注。
  - `analyze_first_board`:相邻板 fdate 间隔 ≥30 分钟视为停机,比较
    停机后首板与其余板的偏差均值与 NG 数;首板样本 <30 判"样本不足"。
  - `analyze_direction`:PrintDirection 列有值则按方向分组比较;
    无值且前后刮刀压力无变化则判"数据未采集",奇偶差仅作参考展示。
- 组装在 pipeline 的 param stage(避免模块环依赖):
  `param_analysis.json` 新增 `process_dimensions` 键。
- 前端"事件与根因分析"页新增"工艺维度体检"区,三张卡按状态着色
  (效应=橙、无效应=绿、样本不足/未采集=灰),清洗周期卡带逐位置
  小条形图。
- 数据侧联动:方向卡与首板卡的"未采集/样本不足"文案直接引用既有
  数据侧需求清单(PrintDirection、CleaningAfterLastBoard)。

## 4. 与知识库的联动(挂起)

体检只描述"维度是否有效应",不直接生成根因候选。当某维度判"存在
效应"且与活动触发/事件在时间上重叠时,才值得把它接入 diagnose 的证据
乘数——那一步等真实出现效应数据再做,避免为不存在的效应预铺规则。
