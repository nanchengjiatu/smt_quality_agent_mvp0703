一次真实的三板连发是怎么处理的（TRG-d1489186，多锡）

触发：C1754_2 在 10:30 / 10:33 / 10:36 连续 3 块生产板判多锡

严格口径：中间无 PASS、复测不计入；触发 ID 由起始板身份哈希派生，重算不变。

观测层把事实算清楚（只描述，不解释）

范围＝单Pad孤立异常（兄弟焊盘与整板上下文均干净）；趋势＝突变型（触发前基线稳定 31.5%）；触发后第 1 块板即恢复，恢复前有 xbackward/xforward 设定变更（记录在案）。

决策四步：真伪 → 嫌疑 → 把握 → 处置

NG 为真 → 提名残锡转印（先验 0.7）/ 参数漂移（0.85）/ 离散变化（0.6）→ 残锡转印签名吻合 ×1.2、单Pad 典型 ×1.1 = 0.924 高；参数漂移范围不典型 ×0.9 = 0.765 → 处置 P2 按首要根因现场确认。

到人眼前：结论、动作、审计链、对话

处置动作直接来自机理卡（清洁钢网底面并复测下一块板，核对擦网周期）；诊断轨迹记录每条规则命中/未命中与算式；对话区可继续追问。

把握度算式（数值来自知识库常量单源，页面自动跟随）
最终把握 = 初始把握 × 症状对不对得上 吻合 ×1.2 / 相反 ×0.7 × 位置典不典型 ×1.1 / ×0.9 × 与擦网周期合不合拍 ×1.15 × 基线是否借自其他机种 ×0.8，封顶 0.95；分档 ≥0.75 高 / ≥0.5 中。
真实例：0.7 × 1.2(签名匹配) × 1.1(范围典型) = 0.924 → 高

交付边界（如实告知）

  • 标定范围：全部阈值与先验标定自单机种、单机台、37 天数据；换机种需用内置回测工具重标定。
  • 机理内容：13 个机理为领域先验，须经产线工艺专家逐条评审后方可作为处置依据（评审前作参考）。
  • 数据侧待接入：SPI 判定阈值（接入后漂移裕度升级为真 Cpk）、PrintDirection / 擦网时刻 / 温湿度（接入后对应机理自动证据点亮）。
  • LLM 对话：启用即有数据出厂（触发上下文发往所选大模型服务商），需数据治理拍板；不启用则纯离线运行。
  • 安全边界：服务无认证，定位为工厂内网工具，依赖网络隔离。
