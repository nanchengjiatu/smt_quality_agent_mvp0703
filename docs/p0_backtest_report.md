  ~ 2024-01-04),含 12 条 NG,分布在 3 个 Pad:C1754_2(多锡触发)、

## 3. 发现

1. **真实提前量存在**:C3706_1(少锡触发 Pad)在多数组合下提前 11~30
   板(≈6~16 小时)被 L2 预警,R997_1 稳定提前 2 板。设计 §2.7 探针
   "全是突发型"的判断被部分推翻——探针只看了末 10 板原始值,EWMA 的
   记忆效应捕捉到了更早的抬升。C1754_2 的"提前量"跨组合抖动剧烈
   (0~235),它是慢性高波动 Pad,其命中更像巧合,不作为证据。
2. **误报率超设计预算 3~12 倍**(预算 ≤1 次/百板,实测最优 3.1):
   结构性原因,不是参数没调好——
   - 92 条序列并联监控,预算均摊到单 Pad 是 0.01 次/百板,任何控制图
     都达不到;
   - 序列存在**台阶式均值迁移**(最长一个 episode 持续 1303 板不回落):
     基线冻结后旧限位永远追不上新常态,这类"误报"其实是真实的过程
     变化,只是 50 板内没恶化成 NG;
   - 误报不集中(61/93 Pad 都有,TOP10 只占 41%),排除个别坏 Pad
     解决不了。
3. **不稳定性与 NG 强相关**:episode 数 TOP 名单里,3 个 NG Pad 及其
   同元件邻居(C3709_1、R997_2)全部在列。预警系统"吵"的地方正是
   质量差的地方——这支持把 L1/L2 沉淀为健康度视图而非弹窗。
4. 换算成运营口径,误报没有看上去可怕:45 板/天 × L3 误报率 3.4% ≈
   **1.5 条页面提示/天**(推荐组合),L2 全量也只有 ~3 条/天。

## 4. 推荐(待拍板)

- **参数:λ=0.2,L=4.0**(粗体行)。理由:L3 误报最低档里唯一保住
  C3706_1 提前量(11 板)的组合;λ=0.2 是 EWMA 惯例中值,回测显示
  结论对 λ 不敏感、对 L 敏感。
- **分级消费方式**(修订设计 §3.2 的呈现层,判据不变):
  - L3 → 页面提示(toast + 预警卡),预期 ~1.5 条/天;
  - L1/L2 → 只进 Pad 健康矩阵与 margin 着色,不打扰;
  - 整板序列(__board__)照常监控,单独一行展示。
- **误报预算改口径**:从"≤1 次/百板"改为"**页面提示 ≤2 条/天**"
  (按当前速率即 ≤4.4 次/百板,L3 门下所有 L≥3.5 组合都满足)。
  原口径是拍脑袋值,回测证明它对 92 路并联监控在物理上不可达。
- 台阶式迁移的长 episode(不回落)在 UI 上应显示为"新常态待确认",
  给一个"接受为新基线"的操作(P3 闭环的一部分),而不是永远红着。
